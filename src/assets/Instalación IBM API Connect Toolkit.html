Instalación IBM API Connect Toolkit

Tiempo estimado de lectura: 2 minutos Editar

Introducción

IBM API Connect Toolkit es una herramienta de desarrollo de APIs, completamente compatible con IBM API Manager para la instalación en equipos locales sin necesidad de tener un IBM DataPower.

Es la herramienta con la que se deberá realizar los desarrollos y las publicaciones a los API Manager de los diferentes entornos. Trabajando dentro de la red de SCA (vía VDI o VPN) se podrán realizar pruebas unitarias e incluso de integración básica con los backends de Desarrollo con los que se comunicará.

Instalación de la herramienta

  1. Descargar e instalar Node.js
  2. Actualizar npm a su última versión Abrir consola de comandos (cmd) con permisos de administrador, y ejecutar la siguiente instrucción: 
    npm install -g npm
  3. Configuración del proxy SCA: Si la instalación se va a realizar desde una máquina situada en la red interna de SCA, es necesario configurar el proxy para poder realizar correctamente la descarga del software. En caso contrario, ++continuar la instalación desde el punto 4++. Configuración del proxy: En la consola de comandos (cmd), ejecutar las siguientes instrucciones sustituyendo ‘username’ y ‘password’ por credenciales de usuario válidas del dominio SCA: 
    npm set proxy http://grupoadeslas.2k3%5Cusername:password@proxy.sca.corp:80
npm config set https-proxy http://grupoadeslas.2k3%5Cusername:password@proxy.sca.corp:80
npm config set registry http://registry.npmjs.org/
  4. Instalar API Connect: 
    npm install -g apiconnect@2.4.11

    Podría darse el caso en el que en la instalación estemos obteniendo un error por no tener previamente instalado Python. En tal taso deberemos instalar este paquete npm previamente:

    npm install python

Getting started

A continuación explicaremos los pasos a seguir para comenzar a trabajar desarrollando APIs

Acceder a la ruda dónde se ha creado la aplicación ejecutar el comando ‘apic edit’ en el directorio en el que deseemos trabajar

$ cd my-notes

Cargar la configuración de la exposición que se desee en función de si se trata de una API interna o externa

apic config:set catalog=apic-catalog://apiconnect.cert.sca.corp/orgs/scainterna/catalogs/desarrollo
apic config:set catalog=apic-catalog://apiconnect.cert.sca.corp/orgs/sca/catalogs/desarrollo

Ejecutar el comando ‘apic edit’

$ apic edit

A continuación se ejecutará el portal de desarrollo para API Connect.

Tip: NOTA: En caso de  aparecer la pantalla de BlueMix solicitando el login en esta plataforma, cerrar y antes de iniciar de nuevo el API Designer ejecutar el siguiente comando: $ set SKIP_LOGIN=true

Para más detalle a cerca de la herramienta, os dejamos la documentación de IBM: API Connect for Developers
